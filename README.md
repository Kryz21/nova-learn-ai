# Novalearn AI

Turn a PDF, DOCX, raw notes, or a YouTube video into structured notes, an auto-graded quiz, and flashcards — generated by Claude.

React + Vite, Tailwind, Framer Motion for the parallax/animated UI, Supabase for auth + database, and a Supabase Edge Function that talks to Claude (so your API key never touches the browser).

## 1. Install

```bash
npm install
```

## 2. Create your Supabase project

1. Go to [supabase.com](https://supabase.com) → New project.
2. In **Project Settings → API**, copy your **Project URL** and **anon public key**.
3. Copy `.env.example` to `.env` and fill them in:

```bash
cp .env.example .env
```

```
VITE_SUPABASE_URL=https://your-project-ref.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-public-key
```

4. In **SQL Editor**, run the contents of `supabase/schema.sql`. This creates the `resources` table with row-level security so users can only ever see their own data.
5. Email/password auth is on by default in Supabase — nothing else to configure for login/signup to work. (You can enable Google/GitHub OAuth later under **Authentication → Providers** if you want.)

## 3. Deploy the Edge Function (where Claude gets called)

Install the Supabase CLI if you don't have it:

```bash
npm install -g supabase
supabase login
supabase link --project-ref your-project-ref
```

Set your Claude API key as a server-side secret (get one at console.anthropic.com — this is the only other API key you need besides Supabase):

```bash
supabase secrets set ANTHROPIC_API_KEY=sk-ant-your-key-here
```

Deploy the function:

```bash
supabase functions deploy generate-content
```

`SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` are automatically available inside Edge Functions — you don't need to set those yourself.

## 4. Run it

```bash
npm run dev
```

Sign up, click **New**, upload a PDF/DOCX/TXT, paste notes, or drop a YouTube link, and it'll generate a study set.

## How it works

```
Browser                          Supabase Edge Function           Claude API
--------                         -----------------------          ----------
Extract text from PDF/DOCX  -->  Verify user's session
(pdfjs-dist / mammoth,            Fetch YouTube captions if
runs client-side)                 sourceType === 'youtube'
                                   Build prompt, call Claude  -->  Generate notes/
                                   Parse JSON response             quiz/flashcards
                                   Insert into `resources`    <--  as structured JSON
                              <--  Return resourceId
Redirect to /resource/:id,
read straight from Supabase
```

- **PDF/DOCX/TXT** are parsed entirely in the browser (nothing leaves the browser except the extracted text).
- **YouTube** transcripts are fetched server-side in the Edge Function via YouTube's public caption track — this only works for videos that have captions (auto-generated is fine), and is unofficial/best-effort since there's no free official transcript API.
- Claude is prompted to return strict JSON (notes sections + key terms, an MCQ quiz with explanations, and front/back flashcards) which is stored as-is in Postgres `jsonb` columns.

## Project structure

```
src/
  components/   StarField (parallax bg), NovaOrbit (hero animation),
                NotesView, QuizPlayer, FlashcardViewer, Navbar, ProtectedRoute
  context/      AuthContext (Supabase auth state)
  pages/        Landing, Login, Signup, Dashboard, NewResource, ResourceDetail
  lib/          supabaseClient.js
  utils/        extractText.js (PDF/DOCX parsing, YouTube ID parsing)
supabase/
  schema.sql                       Run once in the SQL editor
  functions/generate-content/      Edge Function: transcript fetch + Claude call + DB write
```

## Notes / known limits worth knowing about

- YouTube transcript fetching scrapes a public (undocumented) endpoint — it can break if YouTube changes their page structure, and won't work on videos with captions disabled.
- The Claude call uses `claude-sonnet-4-6` with `max_tokens: 8000`; very long source documents are truncated to ~60k characters before being sent — plenty for a chapter/article/video, but a full textbook would need chunking, which isn't implemented yet.
- There's no payment/quota layer — every generation calls the Claude API on your key, so keep an eye on usage if you make this public.
